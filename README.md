# page-ley-archive

## 项目简介

这是一个用于归档和发布独立 HTML 页面的仓库。仓库内保存了多个自包含的网页文件，主要用于记录、展示或分享一些静态内容。这些页面彼此独立，不依赖于任何特定域名或外部服务。

## 目录结构

```
page-ley-archive/
├── pages/          # 存放所有归档的 HTML 页面
├── assets/         # 页面引用的公共资源（如样式、图片、脚本）
├── index.html      # 仓库首页，提供页面索引
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件为一个独立归档单元，文件名通常反映内容主题或创建时间。
- `assets/`：如果页面需要共享资源（如通用 CSS 或 JS 库），可放置于此。
- `index.html`：简单的导航页面，列出当前所有归档页面的链接。

## 页面归档说明

- 所有归档页面均为完整的 HTML 文件，可直接在浏览器中打开。
- 页面内容可能包含文本、图像、样式和脚本，但均内联或引用本地资源，不依赖外部网络。
- 归档不针对任何特定主题或用途，仅作为静态内容的存储与展示。
- 每个页面均保留原始创作时间或归档时间，便于追溯。

## 维护说明

- 本仓库由维护者不定期更新，新增页面时会直接放入 `pages/` 目录并更新索引。
- 如需提交页面归档，可将 HTML 文件放置于 `pages/` 下，并确保文件自包含（无外部链接依赖）。
- 资源文件请放在 `assets/` 中，并确保路径引用正确。
- 不建议修改或删除已有页面，以保持归档的完整性。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/page-ley-archive.git
   ```
2. 直接在浏览器中打开 `index.html` 查看页面列表。
3. 或进入 `pages/` 目录，双击任意 HTML 文件浏览。
